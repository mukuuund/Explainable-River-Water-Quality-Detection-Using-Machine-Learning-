Original Project Goal
=====================
The original idea was to build a system for:
- water quality compliance prediction
- explainable AI using SHAP/LIME
- upstream/downstream pollution analysis
- optional computer vision / visible pollution evidence
- dashboard-based decision support

However, as data realities surfaced, the project was upgraded into a more robust, professional decision-support system prioritizing rigorous data reliability, feature leakage auditing, and real-world operational monitoring over simple academic baselines.

Datasets Used
=============
The project leverages the following primary datasets:

- india_water_quality_preprocessed_phase1.csv (Main ML training dataset)
- NWMP_July2025.csv (Recent operational/demo monitoring data)
- NWMP_August2025_MPCB_0.csv (Recent operational/demo monitoring data)
- NWMP_September2025_MPCB_0.csv (Recent operational/demo monitoring data)
- swq_manual_physical_parameters_cwc_ap_1961_2020.csv (Historical/seasonal baseline methodology data)
- swq_manual_chemical_parameters_cwc_ap_1961_2020.csv (Historical/seasonal baseline methodology data)
- swq_biological_parameter_manual_cwc_ap_2021_2025.csv (Historical/seasonal baseline methodology data)

Limitation: The manual AP historical datasets should not be falsely claimed as Ganga-specific, as they mainly contain Andhra Pradesh river systems.

Compliance Rules
================
Target compliance labels are derived deterministically using the following regulatory thresholds:
- DO is compliant if dissolved_oxygen >= 5
- BOD is compliant if bod <= 3
- pH is compliant if 6.5 <= ph <= 8.5

Since the target labels are entirely rule-derived, ML models using DO, BOD, and pH are naturally expected to perform very well, as they are simply reverse-engineering the regulatory formulas.

Phase 3 Summary — Initial EDA and Baseline ML
=============================================
Two ML experiments were originally planned:

Experiment A (Strict Clean Model): High confidence rows only. Target: strict_compliance_label. Skipped because not enough reliable high-confidence rows existed in model_ready_phase1 for stratified evaluation.

Experiment B (Practical Operational Model): Medium + High confidence rows. Target: available_compliance_label. Trained on 3,171 rows.
Target distribution: 66.3% Compliant / 33.7% Non-Compliant

Initial DecisionTree results were suspiciously perfect:
- Balanced Accuracy: 1.0000
- Non-Compliant Recall: 1.0000
- Non-Compliant F1: 1.0000

This triggered the critical Phase 3.5 sanity audit.

Phase 3.5 Summary — Model Sanity Audit and Leakage Check
========================================================
The audit found the perfect scores were invalid.

Main issue: Hidden target leakage from preserved raw columns such as dissolved_oxygen_mean_mgl, bod_max_mgl, ph_min, dissolved_o2, and other DO/BOD/pH variants which survived into the feature set.

Audit results:
- Feature leakage audit: FAILED
- Duplicate audit: PASSED
- Shuffled-label sanity test: PASSED (Balanced Accuracy: 0.477, ROC-AUC: 0.477)
- Group split with leaked features: FAILED because leaked features were still present

True auxiliary-only ablation revealed realistic performance after explicitly removing all DO/BOD/pH variants:
- Random Split Balanced Accuracy: 0.732
- Random Split F1: 0.649
- Random Split ROC-AUC: 0.807
- Group Split Balanced Accuracy: 0.739
- Group Split F1: 0.657
- Group Split ROC-AUC: 0.787

This proved auxiliary features do contain useful signal, but the original 1.000 results were undeniably due to leakage.

Phase 3.6 Summary — Clean Feature Filtering and Retraining
==========================================================
Phase 3.6 permanently fixed the leakage using explicit canonical feature allowlists. 

Feature variants tested:
1. Core Regulatory Model: dissolved_oxygen, bod, ph
2. Extended Clean Model: core regulatory features + clean auxiliary features
3. True Auxiliary-Only Model: auxiliary features only (no DO/BOD/pH or raw variants)

Group Split Headline Results:
- Core Regulatory Model: DecisionTree, RandomForest, GradientBoosting achieved 1.000 across metrics. Interpretation: expected because the target is derived from DO/BOD/pH.
- Extended Clean Model: DecisionTree, RandomForest, GradientBoosting achieved 1.000 across metrics. Interpretation: expected because core regulatory features are present.
- True Auxiliary-Only Model (GradientBoosting):
  - Balanced Accuracy: 0.698
  - Non-Compliant Recall: 0.528
  - Non-Compliant F1: 0.591
  - ROC-AUC: 0.766

Random Split Auxiliary-Only Result (GradientBoosting):
  - Balanced Accuracy: 0.745
  - Non-Compliant Recall: 0.570
  - Non-Compliant F1: 0.661
  - ROC-AUC: 0.809

Final interpretation: The core and extended models perfectly automate regulatory compliance rules. The auxiliary-only model provides the more interesting academic finding: even without DO, BOD, and pH, secondary indicators can estimate non-compliance risk with realistic performance.

Current Project Interpretation
==============================
The project now has a highly defensible pipeline featuring:
- data reliability checks
- confidence-aware labeling
- leakage audit
- clean retraining
- operational monitoring
- hotspot detection
- alert generation

The project should not overclaim that the ML model independently discovered pollution rules. Instead:
- Core/extended models automate and scale regulatory compliance assessment.
- Auxiliary-only model shows realistic predictive ability when core regulatory parameters are unavailable.
- Operational layer converts predictions into station-level monitoring, hotspots, and alerts.
